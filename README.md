# 🔔 Telegram Alarm Bot — Звонок + Напоминалка

> Бот, который будет звонить тебе в Telegram каждые 2 минуты пока ты не решишь математический пример из 3 класса.
> Потому что кнопка «отложить» — это путь слабаков.

---

> **Лайфхак:** Поставь аватарку аккаунту который звонит — фото солевой альтушки с 3 размером. Гарантируем: когда в 7 утра на экране появится это лицо — ты вскочишь с кровати быстрее чем когда-либо в жизни. Работает лучше любого кофе.

## Зачем это вообще существует?

Потому что обычный будильник можно выключить не просыпаясь. Этот — нельзя.

Бот совершает реальный **P2P-звонок в Telegram** с твоего же аккаунта на твой же аккаунт (или на аккаунт друга, если хочешь испортить отношения). Каждые 2 минуты. До тех пор пока ты не решишь математический пример. Не «нажмёшь кнопку», не «напишешь стоп» — именно **решишь пример**. Да, как в 3 классе.

Звучит жестоко? Ты сам это поставил.

## Возможности

- **☎️ Звонки** — реальные P2P-звонки через Telegram API при срабатывании будильника
- **🧮 Математическая защита** — выключить можно только решив пример (деление, умножение, сложение, комбинации)
- **⏰ Несколько будильников** — до 5 активных будильников одновременно
- **📝 Напоминалки** — текстовые напоминания в заданное время (до 10 штук)
- **🔒 Whitelist** — бот работает только для тех пользователей, которых ты добавил
- **💡 Факт дня** — после победы над будильником бот присылает случайный факт (иногда интересный)
- **🔄 Автовосстановление** — при перезапуске бот вспоминает все активные будильники и напоминалки из БД
- **😤 Нарастающее отчаяние** — сообщения бота становятся всё более настойчивыми с каждым звонком

## Технический стек

| Компонент | Библиотека | Зачем |
|---|---|---|
| Telegram Bot API | `python-telegram-bot` | Принимает команды, шлёт сообщения |
| P2P-звонки | `pyrogram` + `tgcalls` | Реально звонит |
| База данных | PostgreSQL + `SQLAlchemy` (async) | Помнит будильники |
| Планировщик | `APScheduler` | Срабатывает вовремя |
| Шифрование звонков | Diffie-Hellman | Потому что Telegram так устроен |

## Быстрый старт

### 1. Клонировать

```bash
git clone https://github.com/your-username/telegram-alarm-bot.git
cd telegram-alarm-bot
```

### 2. Установить зависимости

```bash
python -m venv venv
source venv/bin/activate  # Windows: venv\Scripts\activate
pip install -r requirements.txt
```

> Если `tgcalls` не ставится — не паникуй, без него бот всё равно звонит (звонок просто не передаёт аудио, но телефон трезвонит).

### 3. Настроить переменные окружения

```bash
cp .env.example .env
nano .env
```

Что заполнить:

```env
BOT_TOKEN=...           # токен от @BotFather
DATABASE_URL=...        # строка подключения PostgreSQL
TELEGRAM_API_ID=...     # с https://my.telegram.org
TELEGRAM_API_HASH=...   # там же
TELEGRAM_SESSION_NAME=pyrogram_session
```

### 4. Создать базу данных

```sql
CREATE USER alarm_user WITH PASSWORD 'alarm_password';
CREATE DATABASE alarm_db OWNER alarm_user;
```

### 5. Авторизовать Pyrogram (один раз)

```bash
python call.py
```

Бот сам спросит номер телефона и код из Telegram — и создаст файл сессии. Это нужно сделать **один раз**. Дальше сессия живёт сама.

> **Это должен быть живой аккаунт**, не бот. Именно с него будут идти звонки. Можно свой запасной номер, можно друга (с его согласия). Главное — аккаунт должен быть в контактах у тех кому будет звонить, иначе Telegram скажет «privacy restricted» и не позвонит.

### 6. Добавить разрешённых пользователей

В `alarm_bot.py` заполни `DEFAULT_ALLOWED_USERNAMES` (юзернеймы без `@`), или добавляй напрямую в таблицу `allowed_users` в БД.

### 7. Запустить

```bash
python alarm_bot.py
```

## Docker — для тех кто не хочет ставить Postgres вручную

Есть `Dockerfile` и `docker-compose.yml`. Postgres внутри, всё само поднимается.

**Важно перед запуском:** сначала авторизуй сессию локально (см. шаг 5 выше), потом залей `pyrogram_session.session` рядом с `docker-compose.yml`. Без этого файла бот поднимется, запустится и не будет звонить. Молча. Как призрак.

```bash
# Собрать и запустить
docker compose up -d

# Посмотреть что происходит
docker compose logs -f bot

# Перезапустить бота
docker compose restart bot

# Всё снести
docker compose down
```

Postgres живёт в volume `postgres_data` — при `down` данные не теряются. При `down -v` — теряются. Не делай `down -v` если не хочешь потерять будильники.

> Если контейнер падает при старте — скорее всего не хватает файла `.env` или сессии. Смотри логи: `docker compose logs bot`.

## Авторизация — что вообще происходит

`call.py` при первом запуске превращается в мастер авторизации:

1. Спрашивает номер телефона (`+7...`)
2. Telegram шлёт код подтверждения
3. Вводишь код — готово, сессия создана

Если у тебя включена двухфакторка — спросит ещё пароль. Не пугайся.

После этого файл `pyrogram_session.session` живёт рядом с ботом и используется для всех звонков. **Не удаляй его** — иначе придётся авторизоваться заново.

> Если запустил бота на VPS и сессии нет — он просто не будет звонить и будет грустно писать об ошибке в логи. Сначала авторизуйся локально, потом заливай сессию на сервер. Или авторизуйся прямо на сервере если там есть терминал.

## Как работают звонки

1. APScheduler срабатывает в нужное время
2. Бот шлёт в чат рандомное сообщение из пула "просыпайся"
3. Тут же отправляется математический пример
4. Pyrogram делает реальный P2P-звонок через Telegram
5. Звонок длится ~30 секунд, потом бот сбрасывает
6. Через 2 минуты — следующий звонок с новым примером и нарастающим отчаянием в тексте
7. Когда пользователь решает пример — цикл останавливается, бот присылает поздравление и случайный факт

> **Важно:** Для звонков нужен **отдельный Telegram-аккаунт** (не аккаунт бота, а живой номер). Именно он будет звонить. `api_id`, `api_hash` и сессия Pyrogram — всё от него.

## Конфигурация

| Константа | По умолчанию | Что меняет |
|---|---|---|

## FAQ

**Q: Это законно?**  
A: Ты звонишь с собственного аккаунта самому себе (или другу с его согласия). Что тут незаконного?

**Q: А если просто не отвечать на звонок?**  
A: Следующий звонок через 2 минуты. И ещё через 2. И ещё. Бот не устаёт.

**Q: Можно использовать чтобы будить друга?**  
A: Если друг добавит тебя в whitelist и авторизует сессию — да. Дружба крепчает через математику в 7 утра.

**Q: Не работает / бот не звонит**  
A: Проверь `.env`. Потом проверь что сессия Pyrogram авторизована (`python call.py`). Потом смотри логи (`journalctl -u alarm-bot -f`). Потом читай README ещё раз.

**Q: Бот пишет что пример правильный, но звонки не останавливаются**  
A: Такого быть не должно, но если происходит — перезапусти бота. Состояние сбросится.

**Q: Можно ли добавить кнопку «отложить»?**  
A: Нет. Это противоречит смыслу существования бота.

## Предупреждения

⚠️ Не ставь будильник на 4 утра «просто чтобы проверить» — работает, не рекомендуем.  
⚠️ Бот не несёт ответственности за испорченные отношения с соседями из-за вибрации телефона.  
⚠️ Убедись что авторизованная Pyrogram-сессия не слетела — иначе бот будет орать в чат, но не звонить. Как менеджер на созвоне.
